# Project: Arangetram Brochure Webpage

*Identity: who / why / scope boundaries. Behavioral specs (FR / NFR) live in `requirements.md`.*

**One-line**: A responsive, bilingual (English/Tamil) webpage hosting the Bharatanatyam Arangetram brochure of Kum. Swapna Ramachandran with an interactive audience feedback section.

**Problem**: A printed Bharatanatyam Arangetram brochure needs a digital companion — one that preserves the visual richness of the print design, is accessible on any device, and allows audience members to leave named feedback that persists to a shared spreadsheet.

**Users**: Event attendees and well-wishers who view the brochure on mobile, tablet, or desktop; the event owner who fills in content and reviews feedback in Google Sheets.

**In scope for v1**:
- 12 brochure sections: cover, school mission + guru message, 7 dance items (Maargam), artist bio, guru bio, orchestra bios, gratitude
- Audience feedback: form (name + text), clickable name list, feedback detail view, Google Sheets persistence via Google Apps Script
- Responsive layout (mobile / tablet / desktop)
- Print-to-PDF (trigger button + print CSS)
- Bilingual: English default, Tamil on-demand toggle (Noto Sans Tamil)
- WCAG 2.1 AA accessibility
- Hosted on GitHub Pages

**Out of scope (explicit non-goals)**:
- Feedback moderation or admin UI
- User authentication
- Analytics / page view tracking
- CMS or database beyond Google Sheets
- Automatic Tamil translation (owner provides Tamil text manually)

**Success criteria**: All 12 brochure sections render correctly across devices; audience members can submit feedback and see their name appear as a clickable link; feedback is written to Google Sheets; Tamil toggle switches font and content slots; print produces clean PDF-quality output; WCAG 2.1 AA checks pass.

**Resolved** *(archived for record)*:
- Feedback is not editable or deletable after submission. (resolved 2026-05-12)
- No moderation step — submitted names appear in the list immediately. (resolved 2026-05-12)
- Tamil translations are pre-generated by LLM (Claude) once English content is finalised, stored as static strings in the i18n data object. No runtime translation API. (resolved 2026-05-12 — see D-001)
- Feedback name list is fetched from Google Sheets on both page load and after each new submission. (resolved 2026-05-12)
