# print

**Purpose**: Provides print-to-PDF capability via a `@media print` CSS layer that hides all interactive elements, controls page breaks across brochure sections, and preserves content layout. Mounts a print trigger button into `#print-trigger-mount` via a minimal JS initialiser. (serves FR-12, NFR-4)

**Public surface**

CSS (`css/print.css`) — the `@media print` block:

Hide rules (elements invisible in print output):
- `#feedback-section` — entire feedback form and name list
- `#i18n-toggle-mount` — language toggle button
- `#print-trigger-mount` — the print button itself (always hidden when printing)

Page-break rules:
- `@page { margin: 1.5cm; size: A4 portrait; }` — standard brochure page margins
- `page-break-before: always` on: `#school-guru`, `#maargam`, `#artist-bio`, `#guru-bio`, `#orchestra`, `#gratitude`
- `page-break-before: always` on each `#dance-item-2` through `#dance-item-7` (items 2–7 each start a new page; item 1 opens on the Maargam section page)
- `page-break-inside: avoid` on: each `#dance-item-N`, `.musician-bio` — prevents a card splitting across pages

JavaScript (`js/print.js`):
- `initPrint()` → `void` — creates `<button id="print-trigger">Print / Save as PDF</button>`, appends it to `#print-trigger-mount`, wires a single `onclick: () => window.print()` handler. No other logic.

**Invariants**
- `#print-trigger-mount` is always hidden under `@media print` — this rule is unconditional and must not be removed.
- `window.print()` is the sole print mechanism — no PDF library, no server-side rendering.
- Print CSS depends exclusively on brochure section IDs (e.g. `#cover`, `#dance-item-1`). A section ID rename in brochure HTML is a breaking change in `print.css` — the two files must be updated together.
- New interactive elements added by other modules are not automatically hidden in print — each module that adds a new interactive element is responsible for adding its corresponding `display: none` rule to `print.css`.
- `css/print.css` is a dedicated file included with `<link media="print">` — it does not live inside the main `style.css`. This keeps print rules isolated and auditable.

**Key choices**
- `[D-006]` `@media print` CSS + `window.print()` — browser-native PDF export; zero dependencies, zero infrastructure. Server-side PDF generation (Puppeteer, wkhtmltopdf) was not considered — overkill for a static brochure page.
- `<link media="print">` for the print stylesheet — the print CSS is never parsed or applied outside print mode, keeping runtime page performance unaffected.

**Non-goals**
- No server-side PDF generation.
- No custom PDF filename (browser controls the save dialog).
- No print preview UI beyond the browser's native dialog.
- Does not manage page headers or footers (browser default headers/footers are acceptable).
- Does not hide elements outside its declared hide-list — other modules own their own interactive elements and must register hide rules here when they add new ones.
