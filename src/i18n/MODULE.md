# i18n

**Purpose**: Manages the English/Tamil language toggle. Owns the static content object keyed by section and field, swaps visible text by reading `data-i18n` attributes defined in the brochure module, and mounts a toggle button into `#i18n-toggle-mount`. Tamil strings are blank by default and populated once the owner finalises English content and requests LLM translation. (serves FR-13, NFR-3)

**Public surface**

JavaScript (`js/i18n.js`):
- `initI18n()` → `void` — mounts the language toggle button into `#i18n-toggle-mount`, calls `setLang('en')` to apply the initial state, enables the button only after `document.fonts.ready` resolves (ensuring Noto Sans Tamil is available before Tamil can be selected). Called once on DOMContentLoaded.
- `setLang(code)` → `void` — accepts `'en'` or `'ta'`; sets `lang="<code>"` on `<html>`; iterates every `[data-i18n]` element and replaces its `textContent` with `CONTENT[key][code]` (falls back to `'en'` if the Tamil string is `""`); iterates every `[data-i18n-alt]` element and replaces `alt`. Idempotent — calling with the current language is a no-op.
- `CONTENT` — module-level `const` object; the single source of truth for all translatable strings:
  ```
  {
    "<sectionId>.<fieldName>": { en: "…", ta: "…" },
    …
  }
  ```
  Keys follow the `<sectionId>.<fieldName>` convention defined in `src/brochure/MODULE.md`. Tamil values are `""` until populated.

Toggle button behaviour:
- Label shows `"தமிழ்"` when English is active; `"English"` when Tamil is active.
- Button carries `aria-label` updated on each toggle: `"Switch to Tamil"` / `"Switch to English"`.
- Mounted at `#i18n-toggle-mount` (owned by brochure).

CSS (`css/i18n.css`):
- `html[lang="ta"] .i18n-text` — activates Noto Sans Tamil font family for all translatable elements.
- `html[lang="ta"] .i18n-text` inherits `font-family: 'Noto Sans Tamil', sans-serif` — no per-element JS font assignment needed.

**Invariants**
- English is always the default on page load — `setLang('en')` fires during `initI18n()` before any user interaction.
- Language state is stored only as `lang` on `<html>` — never in `localStorage` or `sessionStorage`.
- The toggle button is disabled (and carries `aria-disabled="true"`) until `document.fonts.ready` resolves — prevents Tamil being selected before the font is available, which would cause a flash of unstyled Tamil glyphs.
- If a Tamil string in `CONTENT` is `""`, `setLang('ta')` renders the English string for that field — no blank content is ever shown.
- `CONTENT` is the only place translatable strings live — no translatable text is hardcoded in the brochure HTML itself (the HTML carries only the `data-i18n` key as the element's identifier, not the display value).
- The feedback form is excluded from i18n — `#feedback-section` elements carry no `data-i18n` attributes and `setLang` does not touch them.

**Key choices**
- `[D-001]` Tamil strings are pre-generated by LLM at build time and stored statically in `CONTENT` — no runtime translation API. Empty strings fall back to English during development.
- `[D-005]` `html[lang]` attribute + CSS selector for font switching — CSS drives the font change without per-element JS; a single attribute change on `<html>` is sufficient for the entire page.

**Non-goals**
- No auto-detection of browser or OS language preference (`navigator.language` is ignored).
- Supports exactly two languages: English (`en`) and Tamil (`ta`). No infrastructure for additional languages.
- No right-to-left layout support — Tamil is written left-to-right; no `dir` attribute changes.
- Does not translate the feedback form (English-only by design — see `src/feedback/MODULE.md`).
- Does not manage font loading — Noto Sans Tamil is loaded via `<link rel="preload">` in the main HTML `<head>`; i18n only waits on `document.fonts.ready`.
