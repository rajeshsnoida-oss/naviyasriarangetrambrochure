# feedback

**Purpose**: Provides the audience feedback feature — a form collecting visitor name and feedback text, persistence to Google Sheets via a Google Apps Script Web App, and a clickable name list rendered on page load and refreshed after each submission. (serves FR-8, FR-9, FR-10, FR-11)

**Public surface**

JavaScript (`js/feedback.js`):
- `initFeedback()` → `void` — mounts the form submit handler and triggers the initial `loadFeedback()` call; called once by the main page on DOMContentLoaded.
- `loadFeedback()` → `Promise<void>` — GETs all entries from the Apps Script endpoint, renders the clickable name list; called on page load and after each successful submission.
- `FEEDBACK_API_URL` — module-level `const` (string); the deployed Apps Script Web App URL; the only place this value appears.

Apps Script endpoint (`appsscript/Code.gs`) — the cross-system boundary:
- **GET** `<FEEDBACK_API_URL>` → `200 { "status": "ok", "entries": [ { "timestamp": "ISO-8601", "name": "string", "text": "string" } ] }`
- **POST** `<FEEDBACK_API_URL>` with body `name=<string>&text=<string>` (`application/x-www-form-urlencoded`) → `200 { "status": "ok", "entries": [ … ] }` (full updated list returned on every write)
- Error response: `{ "status": "error", "message": "string" }`

HTML section IDs (consumed by the main page):
- `#feedback-section` — the outermost container (hidden in print CSS)
- `#feedback-form` — the `<form>` element
- `#feedback-list` — the `<ul>` rendered with one `<li><a>` per entry
- `#feedback-detail` — the detail panel revealed on name click

**Invariants**
- Feedback is append-only. No edit or delete operations exist at any layer.
- A name appears in the list only after the Apps Script write succeeds — no optimistic rendering.
- `FEEDBACK_API_URL` is defined once as a module-level constant; never inlined elsewhere.
- POST uses `application/x-www-form-urlencoded` (not `application/json`) to avoid the CORS preflight OPTIONS request that Apps Script does not handle. This is a hard constraint on both the JS fetch call and the Apps Script `doPost` handler.
- The Apps Script deployment must be set to "Execute as: Me" and "Who has access: Anyone (no sign-in required)". Any other combination silently breaks the CORS response headers.
- `loadFeedback()` is called on every page load and after every successful submission — entries are never cached in localStorage or sessionStorage.

**Key choices**
- `[D-002]` Google Apps Script as feedback backend — free, native Sheets integration, zero infrastructure; see DECISIONS.md.
- `[D-003]` `application/x-www-form-urlencoded` POST to avoid CORS preflight — Apps Script does not respond to OPTIONS requests; form-encoding keeps it a "simple request" per the Fetch spec.

**Non-goals**
- Feedback form UI is English-only — no Tamil translation of form labels, placeholder text, submit button, or empty-state message.
- No feedback moderation or admin UI.
- No spam prevention or rate limiting (v1 event brochure; acceptable risk).
- No feedback editing or deletion after submission.
- No pagination of the feedback list.
- No email notification on new feedback submission.
- No optimistic UI update before the server confirms the write.
